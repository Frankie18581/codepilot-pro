###### 🚀 阶段一：骨架搭建与基础对话 (The Skeleton)
**目标**：跑通“用户输入 -> LLM -> 输出”的最简链路，搭建好前后端框架，实现基础的流式响应。
**核心价值**：证明你有全栈开发能力，能搭建现代 AI 应用的基础设施。

*   **核心任务**：
    1.  **前端搭建**：使用 Next.js + Tailwind CSS 搭建聊天界面。实现基础的“气泡式”对话 UI，支持 Markdown 渲染（代码高亮）。
    2.  **后端基础**：使用 Python (FastAPI) 或 Node.js 搭建 API 服务。
    3.  **LLM 接入**：接入一个大模型 API (如 OpenAI GPT-4o 或 Claude 3.5)，实现简单的 Prompt 交互。
    4.  **流式输出**：实现 Server-Sent Events (SSE) 或 WebSocket，让文字像打字机一样逐字显示。

*   **技术栈**：Next.js, FastAPI, OpenAI SDK, React-Markdown.
*   **验收标准 (Demo)**：
    *   用户在网页输入“写一个冒泡排序”，能在 2 秒内看到流式输出的代码。
    *   界面整洁，代码块有高亮。
*   **简历亮点**： “基于 Next.js + FastAPI 构建的高性能流式对话前端，延迟低于 200ms。”

###### 🧠 阶段二：引入状态机与多智能体编排 (The Brain - LangGraph)
**目标**：告别简单的“一问一答”，实现 PRD 中的 **多智能体协作 (PM/Arch/Coder/Test)** 和 **状态记忆**。这是你区别于普通学生的关键。
**核心价值**：掌握目前最火的 Agent 编排框架，理解复杂工作流。

*   **核心任务**：
    1.  **定义状态 (State)**：设计 LangGraph 的 `State` 对象，包含 `messages`, `current_step`, `code_artifact`, `test_result` 等字段。
    2.  **构建节点 (Nodes)**：
        *   `Planner`: 分析需求，拆解任务。
        *   `Coder`: 根据任务写代码。
        *   `Tester`: (模拟) 运行代码或检查语法。
    3.  **设计边 (Edges)**：编写条件路由逻辑。例如：如果 `Tester` 发现错误，路由回 `Coder` 修改；如果成功，路由到 `End`。
    4.  **持久化**：使用 SQLite 或 Redis 存储对话状态，支持断点续聊。

*   **技术栈**：LangGraph, LangChain, Redis/SQLite.
*   **验收标准 (Demo)**：
    *   用户输入“做一个猜数字游戏”，系统自动展示：`PM分析 -> Arch设计 -> Coder编码 -> Tester验证` 的全过程日志。
    *   如果代码有语法错误，Agent 能自动尝试修复一次（自我修正循环）。
*   **简历亮点**： “基于 LangGraph 构建状态机驱动的多智能体协作系统，实现需求分析到代码生成的自动化闭环，支持异常自修复。”

###### 💡 阶段三：实现 HIC 核心交互控制器 (The Soul - HIC)
**目标**：落地 PRD 中最具创新点的 **HIC (人性化交互控制器)**。解决“话多”、“幻觉”和“反问生硬”的问题。
**核心价值**：展现你对 LLM 特性的深度理解和控制能力（这是大厂最看重的算法/工程结合点）。

*   **核心任务**：
    1.  **意图复杂度分析器**：在 LLM 生成回答前，先用一个小 Prompt (或小模型) 评估用户问题的“模糊度”和“复杂度”。
    2.  **动态策略路由**：
        *   若 `模糊度 > 阈值`：触发 **单步聚焦澄清** 模式（只问最关键的一个问题，不许长篇大论）。
        *   若 `复杂度高`：触发 **分步执行** 模式（先给大纲，确认后再写代码）。
    3.  **置信度检测与拦截**：在输出前检测模型的“犹豫”信号（如 logprobs 低），主动添加“⚠️ 此部分基于假设”的黄色警告条（如 PRD 所述）。
    4.  **Token 预算控制**：限制单次回复的最大长度，强制模型精简语言。

*   **技术栈**：LangSmith (用于观测), 自定义 Python 逻辑层, Prompt Engineering 高级技巧.
*   **验收标准 (Demo)**：
    *   用户输入模糊需求“做个网站”，系统不再瞎编代码，而是精准反问：“您希望用 React 还是 Vue？主要功能是什么？”（仅一句话）。
    *   当模型不确定时，界面上明显展示“置信度低”的视觉提示。
*   **简历亮点**： “独创 HIC 交互控制引擎，通过动态意图识别与置信度管理，将无效交互轮次减少 40%，显著降低模型幻觉率。”

###### 🔒 阶段四：安全沙箱与代码执行 (The Hands - E2B)
**目标**：让生成的代码真正“跑起来”，并保证安全。
**核心价值**：解决 AI 编程落地的最后一公里——执行与安全，体现工程严谨性。

*   **核心任务**：
    1.  **集成 E2B**：接入 E2B Sandbox API，在云端隔离环境中运行生成的 Python/Node.js 代码。
    2.  **结果反馈闭环**：将沙箱运行的 `stdout` (输出) 或 `error` (报错) 捕获，自动传回给 `Tester` 智能体进行下一轮优化。
    3.  **文件管理**：在沙箱中创建临时文件系统，支持多文件项目的生成和运行。
    4.  **超时与资源限制**：设置代码运行超时（如 10 秒），防止死循环。

*   **技术栈**：E2B SDK, Docker (了解原理即可，E2B 已封装).
*   **验收标准 (Demo)**：
    *   用户让 AI 写一个“计算斐波那契数列”的代码，系统不仅生成代码，还直接在网页右侧窗口运行并显示结果 `0, 1, 1, 2, 3...`。
    *   如果代码死循环，系统自动终止并报错，不会卡死服务器。
*   **简历亮点**： “集成 E2B 安全沙箱实现代码即时执行与验证，构建‘生成 - 运行 - 调试’全自动闭环，确保生产级安全性。”

###### 📈 阶段五：进阶增强 (GraphRAG & 观测) (The Upgrade)
**目标**：处理复杂上下文，引入知识图谱，完善监控。这是加分项，如果时间不够，前四步已经足够强大。
**核心价值**：展示对前沿技术 (GraphRAG) 的探索能力和工程化运维思维。

*   **核心任务**：
    1.  **简易 GraphRAG**：针对特定领域（如某个开源库的文档），构建简单的知识图谱。当用户问到该库的深度用法时，检索图谱而非简单向量匹配。
    2.  **全链路观测**：接入 LangSmith 或 Arize，记录每一次 HIC 的决策、Token 消耗、延迟和用户反馈。
    3.  **评估流水线 (Eval)**：编写脚本，自动测试 50 个案例，统计“代码可运行率”和“用户满意度”。

*   **技术栈**：Neo4j (或 NetworkX), LangSmith, Pytest.
*   **验收标准 (Demo)**：
    *   能回答需要跨文档推理的复杂问题（例如：“A 模块的改动会影响 B 模块的哪个函数？”）。
    *   拥有一个可视化的 Dashboard，展示系统的运行指标。
*   **简历亮点**： “引入 GraphRAG 增强复杂推理能力，建立基于 LangSmith 的全链路可观测与自动化评估体系。”
